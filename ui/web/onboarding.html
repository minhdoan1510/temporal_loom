Features that keep each answer grounded

Temporal Loom is more than a chat surface. Each workspace can bring its own routines, skills, knowledge, appearance, permissions, and connected tools.

Routine feature Built-in Skills Knowledge Base Appearance MCP Servers

Scheduled workflows

Routine feature

Turn repeated team questions into scheduled agent runs, with each routine keeping its timing, status, and execution history in one workspace.

  • Create recurring checks for CS tickets, partner issues, and operational reports.
  • Pause, run, or review a routine without leaving the workspace.
  • Use run history to trace what happened and when the agent last executed.
01

Reusable agent abilities

Built-in Skills feature

Manage SKILL.md bundles inside each workspace, then use AI to generate new skills from team workflows without starting from a blank file.

  • Import, edit, download, or delete reusable skills from one workspace screen.
  • Create skills with AI so domain workflows become structured agent instructions faster.
  • Keep skill descriptions visible so teammates know when each skill should be used.
02

Workspace memory

Knowledge bases for team context

Index Confluence or Markdown sources into Qdrant so agents can retrieve approved workspace knowledge before answering.

  • Track synced knowledge bases, point counts, and last sync date.
  • Keep product rules, runbooks, and team references scoped to the workspace.
  • Use the same context across chats, routines, and investigation handoffs.
03

Workspace control

Appearance and access per workspace

Give admins a single place to set theme mode, workspace accent, and feature permissions without touching code.

  • Switch between light, dark, and system modes for daily use.
  • Choose an accent palette that matches the workspace style.
  • Review tab-level access for sessions, skills, roles, MCP, and knowledge.
04

Connected tools

MCP servers expose the tools agents need

Register authenticated Model Context Protocol servers so agents can query internal APIs, documents, tickets, and evidence sources.

  • Connect workspace-specific APIs, data warehouses, Jira, mail, and search tools.
  • See enabled status, auth mode, tool count, and last sync timestamp.
  • Add or refresh servers without interrupting the active workspace.
05

One workspace example: Lending Claw

Lending Claw is a Temporal Loom workspace configured for Cashloan CS. It turns loan IDs, partner callbacks, and onboarding evidence into visual investigation cards.

Check a loan application

Read onboarding status, partner, amount, contract IDs, and the failing step from one prompt.

Pinpoint partner errors

Connect callbacks and internal functions so the team can see exactly where the flow broke.

Compare partner performance

Create chart-backed readouts for loan count, disbursement amount, and ticket-size differences.

Understand onboarding drop-off

Review the funnel from app open to approval, contract signing, and final disbursement.
